HCI Project Documentation: User Interface Design for Airsoft Player Application

1. Introduction

The purpose of this documentation is to provide an overview of the user interface design for a web/mobile application dedicated to groups of airsoft players. The application aims to enhance the experience of airsoft teams by providing multi-language support for setting up player teams, game type selection, match duration and score, and other preferences. It also offers integration with the internal Bluetooth unit of the weapon replica, providing information such as ammo left, battery capacity and usage, and other maintenance data. Additionally, the system syncs with a smartwatch and a pair of augmented reality glasses to provide contextual information for each player during a match.

2. Interaction Design Patterns

The UI prototype designed in Figma incorporates various interaction design patterns to ensure intuitive and efficient user interactions. Some of the key patterns used in the prototypes are:

  • Navigation Drawer: The navigation drawer pattern is used to provide easy access to various sections of the application, such as team setup, game type selection, match configuration, weapon information, and synchronization options. Users can access these sections by opening the navigation drawer from the app's main screen.
  • Form-based Input: Form-based input is employed for setting up teams, selecting game types, configuring match duration and score limits, and specifying other preferences. This pattern ensures structured data entry, validation, and a consistent user experience.
  • List View: The list view pattern is utilized for displaying teams, game types, and other relevant information. Users can view, add, edit, and delete items in these lists to manage their team composition and game preferences effectively.

3. User Flow

The user flow within the application is designed to provide a seamless experience for airsoft players. The primary user flow includes:

  1. Launching the application: Upon launching the application, users are presented with the main screen, featuring the app's logo and navigation options in the form of a navigation drawer.
  2. Team setup and game type selection: Users can navigate to the team setup screen from the navigation drawer to create, edit, or delete teams. After team setup, they can select the desired game type from the game type selection screen.
  3. Weapon information and Bluetooth integration: Users can access screens displaying weapon-related information and options to sync with the internal Bluetooth unit of their weapon replicas. This allows them to monitor ammo, battery capacity and usage, and perform necessary maintenance tasks.
  4. Smartwatch and augmented reality glasses integration: The application syncs with a smartwatch and augmented reality glasses to provide contextual information during a match. Real-time match data, player stats, maps, and objectives are displayed on the smartwatch and overlaid on the glasses using augmented reality techniques.

4. Affective Factors and Emotion

The design of the application takes into account affective factors and emotions to create a positive user experience. Visual design, feedback and responsiveness, personalization and error handling are some of the aspects considered:

  • Visual Design: The visual design of the application is aimed at evoking excitement and engagement associated with airsoft gameplay. A visually appealing color scheme and graphics are used to create an immersive experience.
  • Feedback and Responsiveness: The application provides immediate and meaningful feedback to user actions, enhancing the sense of control and engagement. Visual and haptic feedback is employed where appropriate.
  • Error Handling: The application employs thoughtful error handling with clear error messages, helping users understand and recover from mistakes or issues.

5. Interaction Types/Paradigms

The airsoft player application incorporates three interaction types/paradigms to cater to different user preferences and provide a diverse user experience:

  • Traditional Interaction: The application utilizes traditional interaction methods such as touch, taps, swipes, and button presses for navigation, form-based input, and general interactions within the user interface.
  • Gestural+Haptic Interaction: The application incorporates gestural+Haptic interaction, allowing users to perform touch gestures such as swiping, pinching, or shaking to interact with the application. Haptic feedback is employed to provide tactile sensations or vibrations, enhancing the user's sense of control and immersion.
  • Mixed Reality: The application integrates with smartwatches and augmented reality glasses to provide contextual information during matches.

6. Conclusion

In conclusion, the user interface design for the airsoft player application aims to provide a comprehensive and engaging experience for airsoft teams. The design incorporates interaction design patterns, ensures a seamless user flow, considers affective factors and emotions, and incorporates three interaction types/paradigms. By considering these aspects, the application aims to enhance the overall airsoft gameplay experience and facilitate smooth and efficient team coordination.
