Comparison of Discord and Skype

Task 1: Delivering an academic presentation

  • Affordances: Both Discord and Skype allow for screen sharing, video conferencing, and real-time messaging, which are useful for delivering academic presentations.
  • Visual variables: Discord offers customizable themes, including dark mode, which can be helpful for reducing eye strain during extended presentations. Skype also allows for screen sharing, but it does not offer customizable themes.
  • Interaction flows: Both Discord and Skype have similar interaction flows, where the presenter can share their screen while also being able to speak and interact with the audience in real-time.
  • Proposed improvement: Discord's customizable themes can be further improved by allowing users to create their own themes and share them with the community. This would give users more control over their visual experience and also promote community engagement.

Task 2: Attending a meeting at late hours

  • Affordances: Both Discord and Skype allow for real-time messaging and voice chat, which can be useful for attending meetings at late hours.
  • Visual variables: Discord's dark mode can be helpful in reducing eye strain during extended meetings. Skype does not offer a customizable theme.
  • Interaction flows: Both Discord and Skype have similar interaction flows, where participants can communicate through real-time messaging and voice chat.
  • Proposed improvement: Both Discord and Skype can be improved by adding a feature that allows users to schedule meetings in different time zones, which can help users attend meetings at more convenient times.

Comparison of visual metaphors and idioms

Discord and Skype both use similar visual metaphors and idioms, such as icons for muting and unmuting microphones, raising hands, and sharing screens. For a text-only interaction, similar visual metaphors and idioms can be implemented using text-based commands, such as /mute to mute the microphone and /share to share the screen.

Cognitive overload in special conditions/contexts

Cognitive overload can occur when there is an excessive amount of information presented to the user, which can lead to difficulty in processing the information. In the context of remote communication applications, cognitive overload can occur during video conferencing, where there are multiple people speaking, as well as when using multiple applications simultaneously.

Three improvements to optimize these applications

  • UI/UX problem identified: Discord and Skype can be overwhelming for new users who are not familiar with the interface. Solution suggested: Both applications can include interactive tutorials that guide new users through the interface and highlight important features.
  • UI/UX problem identified: Discord and Skype can be resource-intensive and may cause lag on low-end devices. Solution suggested: Both applications can include low-resource mode, which reduces the amount of processing power and bandwidth required to run the application.
  • UI/UX problem identified: Discord and Skype do not offer translation services for real-time messaging and voice chat. Solution suggested: Both applications can include translation services for real-time messaging and voice chat, allowing users to communicate with people who speak different languages.

Reconsidering/Adapting Empiric Laws in Multi-Device Interaction

The empiric laws of Fitts, Hick, and Steering are designed to guide the design of user interfaces for single devices. In the context of multi-device interaction, these laws can still be useful but may need to be adapted to account for different interaction flows and the use of multiple devices.

Three new features to improve the experience

  1. Feature: Persistent voice chat rooms. Sketch: Users can create persistent voice chat rooms that remain active even when no one is present. This can be useful for communities that want to maintain a sense of presence even when users are not actively engaging in conversation.
  2. Feature: Customizable hotkeys. Sketch: Users can customize hotkeys for frequently used commands, such as muting and unmuting the microphone or sharing the screen. This can help users save time and reduce cognitive load by reducing the number of clicks required to perform these actions.
  3. Feature: Smart notifications. Sketch: Users can enable smart notifications that filter notifications based on the importance of the message. This can help reduce notification overload and ensure that users are only notified for important messages.
